# Capability Matrix

This matrix separates current scaffold capability from production-ready capability. The root Next.js app now has Supabase-backed slices for registration, approval, memberships, Team Portal reads/theme writes, Team Chat persistence, and provider/mobile foundations. Some UI surfaces still use typed seed data until their screens are wired to the new adapters.

| Capability | Current routes/surfaces | Current state | Production gap |
| --- | --- | --- | --- |
| Multi-theme system | `/team-portal`, `/admin/themes` | Theme presets for soccer, football, baseball, scouts, golf, tennis, swim, and generic. Supabase teams store `themeKey`, mascot, and colors. | Logo asset upload policy and deeper tenant defaults. |
| Theme designer | `/team-portal`, `/admin/themes` | Assigned coach/admin theme writes save to Supabase with authenticated session-derived actor IDs and audit events. Admin console shows all teams, previews, and contrast checks. | Logo upload, richer dark/mobile QA automation, and signed-out/missing-role UX. |
| Parent dashboard | `/parent`, `/parent/rsvp` | Parent-scoped child/team schedule, coach update, RSVP needs, recent media, RSVP button payloads, snack/volunteer openings, and notification preferences load from Supabase for the signed-in user when active guardian/player rows exist. QA parent proof exists via `npm run supabase:qa-users` and `npm run qa:session-proof`. Signed-out and missing-link states show explicit access gates. Route-level tests verify RSVP and preference writes use the verified Supabase session user. | Browser-level live RLS/action tests, provider preference enforcement, and RSVP history UX. |
| Coach dashboard | `/coach`, `/coach/rsvps`, `/coach/parent-replay`, `/api/coach/weekly-update` | Assigned-team RSVP summaries, RSVP reliability patterns, weekly update drafts, weather, snacks, volunteers, and persisted action payloads load from Supabase for the signed-in user when active coach membership rows exist. Weekly updates save as announcements plus pending `team_broadcast` email drafts without provider sends. QA coach proof exists via `npm run supabase:qa-users` and `npm run qa:session-proof`. Signed-out and missing-coach states show explicit access gates. Route-level tests verify snack, volunteer, weather, and weekly-update actions use the verified Supabase session user. | Browser-level live action tests, richer approval queue UI, and production RLS coverage. |
| Admin dashboard | `/admin`, `/admin/health`, `/admin/imports`, `/admin/invites` | Local operations view for teams, registration queue, sponsors, notifications, readiness, imports, and invites. | Organization-scoped admin auth, provider inventory, production audit logs, and approval queues. |

| Google Maps integration | `/team-portal`, `/team-chat`, `/api/field-locations` | Map links exist and field-location API stores Google Maps URLs/embed URLs when key is configured. | Embedded map UI, venue marker management, quota handling, and field layout metadata screens. |
| Team photos/media | `/parent`, `/team-portal`, `/admin`, `/api/media/moderation`, `/api/media/report` | Supabase media reads filter parent/team surfaces to approved media. | Snack schedules | `/parent`, `/coach`, `/team-portal`, `/api/snack-slots/claim` | Signed-in parent and coach dashboards read scoped Supabase snack slots when linked rows exist; claim buttons call the authenticated Supabase claim endpoint. | Volunteer signups | `/parent`, `/coach`, `/team-portal`, `/api/volunteer-signups/claim` | Signed-in parent and coach dashboards read scoped Supabase volunteer roles when linked rows exist; claim buttons call the authenticated Supabase claim endpoint. Route-level action tests assert the verified session user is used. | Role caps, reminders, cancellation flow, approval policies, and browser-level live action tests. |
| Sponsor management | `/admin`, `/api/admin/sponsors` | Admin sponsor CRUD saves to Supabase with authenticated session-derived actor IDs, audit events, pending/active/expired status, placement settings, and queued logo asset rows. | Parent Replay | `/coach/parent-replay`, `/team-portal`, `/api/coach/parent-replay` | Coach selects 2-3 focus areas; deterministic generator creates home activities, translations, memory moment, video recommendation, parent tip, skill cards, and quest. | AI-generated learning plans | `/team-portal`, `/coach/parent-replay` | Deterministic local guidance only; no AI provider connected. | Human-reviewed AI drafts, prompt/eval harness, privacy filters, and provider usage controls. |

## State Legend

- Local scaffold: works in the current app with typed seed data and reducer state only.
- Provider boundary: record/draft exists, but no external send or provider call occurs.
- Production gap: needed before real families, teams, money, or provider delivery depend on the capability.
