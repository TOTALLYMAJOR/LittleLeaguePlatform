# Capability Matrix

This matrix separates current scaffold capability from production-ready capability. The root Next.js app now has Supabase-backed slices for registration, approval, memberships, Team Portal reads/theme writes, Team Chat persistence, and provider/mobile foundations. Some UI surfaces still use typed seed data until their screens are wired to the new adapters.

## State Legend

- Local scaffold: works in the current app with typed seed data and reducer state only.
- Provider boundary: record/draft exists, but no external send or provider call occurs.
- Production gap: needed before real families, teams, money, or provider delivery depend on the capability.
