# Evaluations

Use this folder for repeatable checks of agent behavior and product boundaries.

Start with `docs/evaluation-plan.md`.

Recommended future files:

- `access-leakage.md`
- `unsafe-provider-send.md`
- `registration-match-risk.md`
- `child-name-exposure.md`
- `retention-boundary.md`

Each evaluation should include:

- Scenario.
- Actor role.
- Prompt or user action.
- Allowed context.
- Expected answer or behavior.
- Failure examples.
